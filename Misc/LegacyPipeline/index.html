    Legacy Pipeline

    Description

    The legacy pipeline was the first pipeline that was ever released with MollyEngine since it was based upon the original rendering system. This means that the legacy pipeline is quite deprecated and inefficient when it comes to resources since the very early builds of MollyEngines in which this pipeline was based upon where not designed as carefully as the newer systems and pipelines.

    How does it work

    The rendering does the following, in the following order.

    1. Clears the screen and fills it with the background color.

    2. Updates the camera position.

    3. Creates a list of all the elements in the game it will need to render.

    4. Due to the fact that a Shape2D is rendered differently than a Sprite2D and that the rendering list is made out of GameObjects we need to identify every GameObject who's type is null/undefined. So we do exactly that in step 4.

    5. Loops through our list of GameObjects we need to render and checks whether or not the GameObjects priority is equal to the currentRenderingPriority. Every time the loop is completed the currentRenderingPriority is incremented and any items that have been rendered are removed from our list of GameObjects we need to render.

    Notes

    The rendering algorithm used by MollyEngine is far from perfect and some flaws do exist. For example if you have 2 objects(a and b) and objects a rendering priority is 0 while objects b rendering priority is 100 the loop will run 100 times until the currentRenderingPriority reaches 100.
